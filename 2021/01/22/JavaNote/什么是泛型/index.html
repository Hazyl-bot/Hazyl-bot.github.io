什么是泛型

在C语言中，需要可变长度的数组时我们通常使用malloc,calloc函数（链表），将计算好的空间分配给某一个指针或数组，在C++中使用vector，而在JAVA中，我们使用ArrayList来定义一个有序的可变长度的数组。


但是，例如C语言中申请储存n个int数据的空间时需要malloc(sizeof(int)*n)，JAVA有没有类似的表达呢，如果没有它又怎么区分不同变量需要的不同命名空间呢?
第一种方法：直接读入，读取时强制转型即可，但是这样很容易出错，误转型或者读出奇怪的数据等
第二种方法：以String为例，单独定义一个StringArrayList，需要哪个数据类型，就使用相应的列表型。这样很好操作也很好实现，但JAVA中的类千千万万，这些类型数组是定义不完的，重写方法也很繁杂。

在 Java 中增加范型类之前， 泛型程序设计是用继承实现的。ArrayList 类只维护一个 Object 引用的数组： public class ArrayList // before generic classes
private Object[] elementData;
public Object get(int i) { . . , }
public void add(Object o) { . . . }
这种方法有两个问题。当获取一个值时必须进行强制类型转换。
ArrayList files = new ArrayListO；
String filename = (String) files.get(O);
此外，这里没有错误检査。可以向数组列表中添加任何类的对象。
files,add(new File(“. .
对于这个调用，编译和运行都不会出错。然而在其他地方，如果将 get 的结果强制类型 转换为 String类型， 就会产生一个错误。 泛型提供了一个更好的解决方案： 类型参数（type parameters)。ArrayList 类有一个类型 参数用来指示元素的类型： ArrayList files = new ArrayList(): 这使得代码具有更好的可读性。人们一看就知道这个数组列表中包含的是 String 对象。//JAVA核心技术卷

小结：
1、泛型就是编写模板代码来适应任意类型；


2、泛型的好处是使用时不必对类型进行强制转换，它通过编译器对类型进行检查；


3、注意泛型的继承关系：可以把ArrayList<Integer>向上转型为List<Integer>（T不能变！），但不能把ArrayList<Integer>向上转型为ArrayList<Number>（T不能变成父类）。
4、除了ArrayList<T>使用了泛型，还可以在接口中使用泛型。
顺带一提：如果想要在对象数组中基于某个值进行排序，则需要对这个类实现Comparable这个泛型接口
