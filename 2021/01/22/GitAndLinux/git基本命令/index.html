绑定本地仓库到GitHub

我们在使用GitHub之前已经写了许多编程项目，接下来我们把这些项目放到GitHub上。

操作步骤

首先我们要在GitHub上创建一个跟项目文件夹同名的空仓库。注意创建时不要勾选自动生成readme.md，这样可以避免代码冲突
然后在项目文件夹下使用git init。注意这里如果初始化错了，比如初始化在上一级，那么就需要删除上一级的.git 文件然后重新操作
然后使用git remote -v查看当前仓库是否已被绑定，期望值是空白。
然后我们使用git remote add origin 仓库地址绑定远程仓库，如果绑定错误，就用git remote remove origin移除绑定重新来过。

报错解决

以上是GitHub友好的新手提示

push

push简单的说就是把你的本地代码上传到GitHub

第一次push

  1. git add -a

  2. git commit -m “你想要补充的评论和警告”

  3. git push origin master

  4. 如果不想push到master分支的话第3步使用git push origin 分支名

    第2-n次push

    第2+次push一般都是把网上的项目代码进行修改，所以这里从clone开始

  5. git clone url （url是项目的GitHub地址

  6. 编辑你的项目

  7. 如果你对项目代码同时进行了删除和增加的话需要：git pull

  8. 和第一次一样git commit -m “input your commit”

  9. git push origin master

    Git分支

    我们有的时候需要备份一些重要的功能模块，但是又不希望他跟项目代码混在一起，这时候我们可以使用git branch 相关指令

    基本指令

    创建分支 git branch xxx
    切换到某分支 git checkout xxx
    创建并切换到某分支 git checkout -b xxx
    直接下载某仓库的某分支 git clone url xxx
